# 04 風險與治理原則

## 文件目的

本文件定義 FALO IM / LINE Lab 的研究邊界。

本專案會接觸 IM 溝通資料，而 IM 資料通常包含個資、商業機密、客戶資訊、員工互動與決策脈絡。因此初期就必須把治理邊界寫清楚。

## 核心原則

一句話：

> 只處理使用者本人授權、本人可見、可合法取得的資料，並把 AI 結果維持在可覆核狀態。

## Observer-only 安全邊界

本專案的地端 Agent 預設角色是「觀察者」，不是 LINE 的修改工具。

Observer-only 代表：

- 只讀取：可以掃描路徑、列出檔案、讀取使用者授權的匯出檔或可見資料。
- 只觀察：可以建立外部事件流、OCR 結果、Markdown 筆記、AI Note 素材。
- 不修改：不修改 LINE app、LINE 本機資料夾、LINE 資料庫或快取。
- 不更新：不寫回任何 LINE 內部資料、設定、Session 或資料庫。
- 不刪除：不刪除 LINE 檔案、快取、訊息、媒體或帳號資料。
- 不偽裝：不冒充 LINE client，也不替使用者送出訊息。

換句話說，FALO IM Agent 可以在 LINE 外部建立「觀察紀錄」與「AI 工作流資料」，但不能改變 LINE 原本的資料狀態。

## 允許研究的範圍

允許：

- 使用者本人裝置。
- 使用者本人帳號。
- 使用者明確授權。
- LINE Desktop 畫面上已顯示的內容。
- 使用者主動匯出的聊天紀錄。
- 使用者主動轉傳給 Bot 的訊息。
- OS 通知或 Accessibility API 在授權後可取得的內容。
- read-only 掃描 LINE 本機資料夾的路徑、檔案類型、大小、修改時間。
- 對 LINE 原始資料夾外部的 clone / snapshot 做 read-only 解密研究。
- 本機 PoC、教學示範、流程驗證。

## 不允許的範圍

不允許：

- 破解 LINE。
- 在 LINE 原始資料夾內解密、修復、轉檔或寫入。
- 繞過 LINE 存取控制。
- 竊取 Token / Cookie / Session。
- 讀取他人未授權帳號。
- 隱藏式監控員工。
- 未告知群組成員就長期蒐集聊天內容。
- 把 AI 推論直接當成法律或稽核最終證據。
- 修改、更新或刪除 LINE 本機資料。
- 寫回 LINE 內部資料庫或快取。

允許與禁止的差異在於：可以研究 clone 的格式與解密方法，但不能改變 LINE 原始資料，也不能取得或濫用登入憑證。

## 主要風險

| 風險 | 說明 | 控制方式 |
|---|---|---|
| 個資風險 | LINE 訊息可能包含姓名、電話、地址 | 最小化保存、去識別化、權限控管 |
| 商業機密 | 對話可能包含報價、合約、客戶資訊 | 本機處理、加密保存、限制輸出 |
| 誤判風險 | OCR 或 AI Parser 可能錯讀 | 信心分數、人工覆核、保留原始參照 |
| 監控風險 | 工具可能被誤用成員工監控 | 明確授權、可停止、可查看紀錄 |
| 證據風險 | AI 摘要可能不等於原始事實 | 摘要與原文分離、來源可追溯 |
| 合規風險 | 不同企業與場景法遵要求不同 | 導入前做資料盤點與同意流程 |

## 資料最小化

初期 PoC 應遵守資料最小化：

- 只截取必要區域。
- 只保存必要文字。
- 圖片可選擇不長期保存。
- 敏感資訊可遮罩或 hash。
- 預設本機儲存。
- 預設不自動上雲。

## 同意與告知

如果進入企業場景，至少要說清楚：

- 工具會觀察什麼資料。
- 資料會保存多久。
- 誰可以看結果。
- AI 會做哪些萃取。
- 哪些結果需要人工確認。
- 使用者如何暫停或刪除資料。

群組場景尤其要注意：

- Bot 在群組中應明確可見。
- 地端觀察若涉及群組內容，需確認企業政策與成員告知。
- 不應把私人聊天內容混入企業知識庫。

## AI 輸出治理

AI Parser 的輸出應分層：

| 層級 | 說明 | 是否可直接採用 |
|---|---|---|
| Raw Event | 原始事件或原始文字 | 可作來源參考 |
| AI Draft | AI 萃取的任務、風險、摘要 | 不可直接當最終事實 |
| Human Reviewed | 人工確認後的結果 | 可進入工作流 |
| Audit Evidence | 經整理、留痕、可追溯資料 | 需符合企業稽核規範 |

## 稽核留痕

每個事件應盡量保留：

- 事件 ID。
- 來源類型。
- 擷取方法。
- 擷取時間。
- 處理時間。
- AI 模型或 Parser 版本。
- 信心分數。
- 人工覆核人。
- 人工覆核時間。
- 最終採用狀態。

## 對外說法

建議使用：

- 授權地端觀察 Agent。
- IM Intelligence。
- LINE 工作訊息事件化。
- AI 工作流入口。
- 可追溯的任務與決策萃取。

避免使用：

- 偷讀 LINE。
- 破解 LINE。
- 監控員工聊天。
- 自動抓所有聊天紀錄。
- 無感蒐集。

## 第一階段治理結論

本階段只做 PoC，不追求完整產品化。

PoC 應證明：

1. 使用者可控制資料來源。
2. 系統不碰帳密、Token、Session。
3. 事件轉換過程可追溯。
4. AI 結果需要人工覆核。
5. 輸出適合教學、展示與後續治理討論。
