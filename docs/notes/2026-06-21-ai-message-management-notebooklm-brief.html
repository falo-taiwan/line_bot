AI Message Management × NotebookLM 架構備忘

把 LINE Monitor、官方帳號與匯出檔取得的訊息，整理成本地資料倉庫，再轉成 NotebookLM 可問答的來源文件。

一、核心定位

這不是先做自動客服，而是先做「AI 訊息管理助理」。

LINE 資料進來後，不直接丟給 AI 猜答案，而是先經過本地 ETL、中控倉庫與來源檔整理。NotebookLM 只負責讀整理後的 source files，提供摘要、問答、追溯與教材化。

LINE 資料來源
  ↓
本地中控 ETL
  ↓
本地資料倉庫
  ↓
NotebookLM source files
  ↓
NotebookLM 問答 / 摘要 / 教材化
重要觀念：本地資料倉庫是 source of truth，NotebookLM 是可問答的知識副本。

二、資料來源

來源 說明 用途
LINE Official Account / Bot 官方帳號進駐群組後收到的訊息 即時監控、單群 / 多群摘要
LINE Monitor 本地授權觀察或其他合法監控來源 本地事件化、PoC、教學展示
LINE 匯出 txt 使用者手動匯出的歷史對話紀錄 歷史補資料、parser 測試、案例建立

第一階段要守住邊界：接收、保存、分類、摘要可以自動；對外回覆與推播先由人工確認。

三、本地中控程式

中控程式的定位是 ETL + Local Data Warehouse + 操作中控台。它不是把資料鎖進黑盒，而是把資料整理成人和 AI 都能使用的檔案。

Extract接收 LINE Monitor、官方帳號 webhook、匯出 txt。
Transform解析、標準化、依日期拆檔、產生 latest memo。
Load輸出到本地倉庫、NotebookLM incoming folder 或 command package。
Audit保存原始檔、manifest、hash、upload log。
Recovery程式壞掉時，人工仍可打開檔案接手。
Governance控制哪些資料能送 AI，哪些要遮蔽或覆核。

五、NotebookLM Source 策略

NotebookLM 裡面不只放日期記錄檔，還要放固定指示檔與最新狀態 memo。

檔案 角色 替換方式
00_*_instruction.md 告訴 NotebookLM 如何理解資料本 少改，必要時替換
01_*_latest_memo.md 最新狀態、待辦、目前重點、回答原則 永遠同名替換
LINE_*_YYYY-MM-DD.md 每日訊息紀錄與摘要 當日可替換，過去逐步凍結

六、Latest Memo + Daily Source

這是目前的核心策略：最新狀態 Memo + 每日來源檔。

來源檔 回答的問題
latest_memo.md 現在狀態是什麼？哪些事情待處理？AI 回答要遵守什麼？
daily_YYYY-MM-DD.md 那一天發生了什麼？誰說了什麼？細節在哪裡？

回答「目前、最近、今天」時，優先看 latest memo；回答「某一天發生什麼」時，再查對應日期檔案。

七、Prepared Prompt

準備好的 prompt 會讓 NotebookLM 更可靠，因為它明確知道指示檔、latest memo 與日期檔案的關係。

請你依照本資料本中的檔案邏輯回答問題。

請先閱讀指示檔案，理解：
1. latest_memo 是最新狀態與目前指令。
2. 日期檔案是每日歷史紀錄與細節來源。
3. 回答目前狀態時，請優先參考 latest_memo。
4. 回答特定日期事件時，請回查對應日期檔案。
5. 如果 latest_memo 與日期檔案有差異，請明確說明差異。

現在請幫我整理：
一、最新狀態
二、主要討論主題
三、待處理事項
四、可轉成教材的內容
五、需要人工判斷或補充的地方
六、你參考了哪些檔案

九、人工備援

即使程式壞掉，流程也不能停。

  1. 打開本地資料倉庫。
  2. 進入 notebook_sources/群名/。
  3. 找到 instruction、latest memo、今天的 daily file。
  4. 手動上傳或替換到 NotebookLM。
  5. 在 upload_log.csv 或人工紀錄補上操作時間與操作者。
核心原則：自動化可以停，但資料必須耐用、可讀、可搬移、可人工接手。

十、下一步

  2. 先寫 00_instruction.md。
  3. 再寫 01_latest_memo.md。
  4. 補 2 到 3 天的 LINE_*_YYYY-MM-DD.md。
  5. 準備固定 prompt。
  6. 測試 NotebookLM 是否能理解「指示檔 + latest memo + 日期檔」邏輯。
